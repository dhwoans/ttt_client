# 📋 턴 제한 시간(Countdown) 기능 
## 1. 기능 개요
각 플레이어의 턴에 제한 시간을 부여하여 게임의 지연을 방지하고 긴장감을 유지합니다. 시스템은 실시간으로 남은 시간을 추적하며, 시간 초과 시 해당 플레이어를 즉시 패배 처리합니다.

---

## 2. 핵심 동작 흐름 (Core Logic)

| 단계 | 이벤트 | 시스템 동작 | 상태 |
| :--- | :--- | :--- | :--- |
| **1** | **턴 시작** | 타이머를 **10초**로 초기화하고 즉시 카운트다운 시작 | `RUNNING` |
| **2** | **시간 감소** | 100ms 단위로 실제 경과 시간을 계산하여 초 단위로 UI 업데이트 | `RUNNING` |
| **3** | **착수(성공)** | 제한 시간 내 플레이어가 수를 두면 타이머 즉시 중지 및 초기화 | `IDLE` |
| **4** | **시간 초과** | 0초 도달 시 `onComplete` 이벤트 발생 → 현재 플레이어 **강제 패배** | `EXPIRED` |
| **5** | **게임 종료** | 승패 결정 시 모든 타이머 프로세스 강제 종료 및 리소스 해제 | `IDLE` |

---

## 3. 세부 기능 (Detail)

### 3.1 카운트다운 정책
* **기본 시간:** 턴당 **10초** (10,000ms).
* **정밀도 유지:** 브라우저 성능 저하에 따른 오차를 방지하기 위해 `Date.now()` 기반의 **실제 경과 시간** 보정 방식을 채택합니다.
* **일시 정지 조건:** 게임 결과 팝업, 네트워크 연결 끊김 시 타이머 일시 정지.

### 3.2 UI/UX 피드백 (Visual Feedback)
| 잔여 시간 | UI 연출 | 의도 |
| :--- | :--- | :--- |
| **10s ~ 4s** | 기본 색상 (Black/White) | 안정적인 정보 전달 |
| **3s ~ 1s** | **빨간색(Red) 전환 + 1.1배 확대 애니메이션** | 시각적 경고 및 긴박함 조성 |
| **0s** | "Time Out!" 텍스트 표시 | 결과 확정 피드백 |

### 3.3 예외 상황 처리 (Edge Cases)
* **탭 전환:** 탭 복귀 시 실제 경과 시간을 계산하여 잔여 시간 강제 동기화.
* **레이스 컨디션:** 로컬 모드는 착수 우선, 멀티 모드는 서버 판정 우선.

---

## 4. 인터페이스 명세 (API)

### Props (Input)
* `initialSeconds`: 시작 시간 (기본값 10)
* `isPaused`: 타이머 일시 정지 여부
* `onComplete`: 0초 도달 시 실행할 콜백 함수

### Exposed Methods
* `start()` / `stop()` / `reset(time)`
